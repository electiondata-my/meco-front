## Use Case

Use these endpoints to build a candidate lookup feature, similar to the one offered on our [Candidates dashboard](https://www.electiondata.my/candidates). A user searches for a candidate by name, selects one, and gets back every election result for that candidate, spanning both general elections (Parliament and State) and by-elections.

Just follow the 3 simple steps below, and you'll be up and running!

## Step 0: Generate token

## Step 1: Get list of candidates

To let users search for a candidate, you first need a list of every candidate who has ever contested an election, and their corresponding unique ID.

<EndpointBadge method="GET" url="https://api.electiondata.my/v1/candidates/dropdown" />

This endpoint takes no parameters. To improve your product's performance, we recommend fetching this once and caching it, or even bundling it at build time. This is because the list is only updated after the nomination day for a particular election (weeks before the actual voting day), after which it is frozen until the next election's nomination day, so there is really no reason to fetch it more than once per session or per build (if your page is static).

## Step 2: Get single Candidate

Once the user has selected a candidate, pass their `id` to retrieve their election results.

<EndpointCard method="GET" url="https://api.electiondata.my/v1/candidates" params={[
  {
    name: "id",
    type: "string",
    required: true,
    description: <>Unique identifier derived from the dropdown.</>
  }
]} />

The `id` parameter is mandatory; requests without this parameter will return a `400 Bad Request` error, while requests with an invalid `id` will return a `404 Not Found` error.

  { name: "party_uid", type: "string", description: "Unique party identifier e.g. '021-DAP'." },
  { name: "coalition", type: "string", description: "Coalitional affiliation of the party, if any (e.g. 'PH')." },

  { name: "votes", type: "integer", description: "Total votes received by the candidate." },
  { name: "votes_perc", type: "float", description: "Percentage of total valid votes gained by the candidate." },
  { name: "result", type: "string", description: "One of: 'won', 'won_uncontested', 'lost', or 'lost_deposit'." },
]} />
