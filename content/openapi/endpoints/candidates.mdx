## Use Case

Use these endpoints to build a candidate lookup feature—something like the [Candidates dashboard](https://www.electiondata.my/candidates) on this site. A user searches for a candidate by name, selects one, and gets back every election result for that candidate, spanning both general elections (Parliament and State) and by-elections.

There are two endpoints involved in this workflow, and they are designed to be used in sequence.

## Step 1: List of Candidates

To let users search for a candidate, you first need a list of all candidates and their unique IDs.

<EndpointBadge method="GET" url="https://api.electiondata.my/v1/candidates/dropdown" />

This endpoint takes no parameters. We recommend fetching this once and caching it—the list only changes when new election data is added, so there is no need to query it on every user interaction.

<TabCode

A successful `200 OK` response returns a JSON array of every candidate in the database, in descending order of the number of elections they have contested. Each object in the response contains:

  { name: "id", type: "string", description: "Unique candidate identifier. Use this in Step 2." },
  { name: "name", type: "string", description: "Candidate's full name." },
  { name: "c", type: "integer", description: "Total contests stood in." },
  { name: "w", type: "integer", description: "Total contests won." },
  { name: "l", type: "integer", description: "Total contests lost." },
]} />

## Step 2: Specific Candidate

Once the user has selected a candidate, pass their `id` to retrieve their election results.

<EndpointCard method="GET" url="https://api.electiondata.my/v1/candidates" params={[
  {
    name: "id",
    type: "string",
    required: true,
    description: <>Unique identifier derived from the dropdown.</>
  }
]} />

The `id` parameter is mandatory; requests without this parameter will return a `400 Bad Request` error, while requests with an invalid `id` will return a `404 Not Found` error.

<TabCode

A successful `200 OK` response returns a JSON array of election results, ordered from most recent to oldest. Each object in the response contains:

  { name: "party_uid", type: "string", description: "Unique party identifier e.g. '021-DAP'." },
  { name: "coalition", type: "string", description: "Coalitional affiliation of the party, if any (e.g. 'PH')." },

  { name: "votes", type: "integer", description: "Total votes received by the candidate." },
  { name: "votes_perc", type: "float", description: "Percentage of total valid votes gained by the candidate." },
  { name: "result", type: "string", description: "One of: 'won', 'won_uncontested', 'lost', or 'lost_deposit'." },
]} />
