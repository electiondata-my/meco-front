## What is it?

This is the dynamic, programmatic interface to the datasets made available and visualised on the site. All data is dedicated to the public domain (marked with **CC0 1.0 Universal**)—so you are free to use this API for any purpose, commercial or otherwise, with or without attribution.

The API is free to access. There is no rate limit and therefore no premium tier. All you need is an API key, which you can generate in seconds from the [API Console](/console).

## Do I need this?

ElectionData.MY offers two methods to programmatically access the data: Direct access to entire files via the open data lake (use the links provided via the [Data Catalogue](/data-catalogue)), or this open API.

The API and the data lake serve different use cases. To put it simply, use this API if you want to build public-facing products and don't want to build/host your own backend; use the data lake if you want the entire dataset for analysis, local use, or to spin up your own backend (although you will hopefully first explore this API, which has been built to industry-grade specifications).

To put it more thoroughly, use the API if…

- You are building a dynamic application that queries election data on demand—e.g. a candidate lookup tool or a live election dashboard.
- You are embedding election data into another product, such as a news app, civic platform, or data pipeline.
- You need a stable, versioned interface with consistent response schemas, rather than writing custom parsing logic against raw files that may change structure over time.

## Base URL

All API requests are made to the following base URL:

```
https://api.electiondata.my
```

Append the endpoint path, version, and query parameters to construct a full request URL. For example: `https://api.electiondata.my/v1/candidates?id=YQJ5S`.
