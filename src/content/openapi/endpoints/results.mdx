## Use Case

The `/results` endpoint is designed to be used **together** with four other APIs, not in isolation. Each of those APIs returns a summary list of contests; `/results` gives you the complete details for a single contest:

- **Seats**: returns a list (with summary details) of every contest held for a particular constituency; use `/results` to get the complete details for a specific contest.
- **Candidates**: returns a list (with summary details) of every contest a candidate has stood in; use `/results` to get the complete details for a specific contest.
- **Elections**: returns (among other things) a list of every seat contested in a general election; use `/results` to get the complete details for a selected seat.
- **By-Elections**: returns a list of all by-elections ever held; use `/results` to get the complete details for any one of them.

In short, you should use this endpoint to allow users to zoom into a specific contest, similar to what is done on our [Seats](https://www.electiondata.my/seats), [Candidates](https://www.electiondata.my/candidates), [Elections](https://www.electiondata.my/elections), and [By-Elections](https://www.electiondata.my/byelections) dashboards.

Just follow the 2 simple steps below, and you'll be up and running!

## Step 1: Get params

The `/results` endpoint mandates three parameters, all of which come directly from the response of the four companion APIs listed above. In each case, the mandatory parameters you need (`seat`, `state`, and `date`) are already present in the objects returned by those APIs. Pass them straight through—no transformation needed.

<EndpointCard method="GET" url="https://api.electiondata.my/v1/results" params={[
  {
    name: "seat",
    type: "string",
    required: true,
    description: "Constituency identifier, e.g. 'P.001 Padang Besar' or 'N.01 Banggi'. Taken directly from the seat field in the companion API response."
  },
  {
    name: "state",
    type: "string",
    required: true,
    description: "State name, e.g. 'Perlis' or 'Sabah'. Taken directly from the state field in the companion API response."
  },
  {
    name: "date",
    type: "string",
    required: true,
    description: "Polling day in ISO 8601 format (YYYY-MM-DD), e.g. '2022-11-19'. Taken directly from the date field in the companion API response."
  }
]} />

All three parameters are mandatory. Missing any one will return a `400 Bad Request` error. A `date` that is not in `YYYY-MM-DD` format will also return `400`. A valid combination that does not correspond to a real contest will return `404 Not Found`.

A successful `200 OK` response returns a JSON object with two keys: `ballot` and `stats`.

---

**ballot**: an array of every candidate who stood in the contest, ordered by votes received (descending):

<FieldTable fields={[
  { name: "name", type: "string", description: "Candidate's full name." },
  { name: "party_uid", type: "string", description: "Unique party identifier (e.g. '004-PAS')." },
  { name: "party", type: "string", description: "Acronym of the candidate's party (e.g. 'PAS')." },
  { name: "coalition_uid", type: "integer", description: "Numeric coalition identifier (0 if the party contested independently)." },
  { name: "coalition", type: "string", description: "Coalition name (e.g. 'PN'). 'ALONE' if the party contested independently." },
  { name: "votes", type: "integer", description: "Total votes received (zero for uncontested wins)." },
  { name: "votes_perc", type: "float", description: "Share of valid votes as a percentage (null if uncontested)." },
  { name: "result", type: "string", description: "One of: 'won', 'won_uncontested', 'lost', or 'lost_deposit'." },
]} />

---

**stats**: an array containing a single object with aggregate statistics for the contest:

  { name: "majority_perc", type: "float", description: "Majority as a percentage of total valid votes (null if uncontested)." },
]} />
